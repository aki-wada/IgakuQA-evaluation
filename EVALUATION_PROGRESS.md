## 2. 主要な発見・考察

### 2.1 max_tokensの重要性（reasoningモデルでは最重要パラメータ）

- gpt-oss-120b GGUF: mt=50(29-33%) → mt=200(82-85%) → **mt=1024(85-90.7%)**
- gpt-oss-120b MLX 8bit: mt=50(26-30%) → **mt=1024(85-92.0%)** ← **全モデル最高スコア⭐⭐⭐**
- gpt-oss-20b: mt=50(26-32%) → mt=200(68% 案Bのみ) → **mt=1024(74-77.3%)** ✓合格ライン達成
- minimax-m2.5: mt≤1024(34.7%) → **mt=4096(77.3%)** ✓合格
- qwen3-235b-a22b-thinking-2507: mt不足(70.7%) → **mt=8192(89.3%)**
- gpt-oss-120b: reasoningトークンがmax_tokensを消費 → contentに回答が出力されない
- mt=50: 空回答43-49件 → 200: 4-6件 → **1024: 0件**（完全解消）
- medgemma-27b: max_tokens=200で逆に複数選択肢を出力し抽出失敗
- **結論**: reasoningモデルにはmax_tokens≧1024を推奨、非reasoningモデルは50-200で十分

### 2.2 日本語ファインチューニングの効果

- **llama-3.3-70b 日本語FT比較**: Swallow(81.3%) >> base(68.0%) > Shisa(61.3%) → FT手法で20%の差
- **Swallow効果**: llama-3.3-70b(68.0%) → swallow-70b(81.3%) で**+13.3%**向上
- **Swallowスケーリング**: 8B(53.3%) → 70B(81.3%) で**+28.0%**向上
- **Shisa日本語FTは逆効果**: llama-3.3-70b(68.0%) → shisa-v2(61.3%) で**-6.7%**低下
- **EZO日本語FT効果**: gemma-3-12b(54.7%) → ezo2.5(60.0%) で**+5.3%**向上
- **GPT-OSS-Swallow-20B**: gpt-oss-20b(~71%) → Swallow FT → **77.8%で合格**（+6.8%）
- **結論**: 日本語FTが必ずしも改善にならない。FT手法の質が重要

### 2.3 スケーリング効果

- **qwen3シリーズ**: 4B(54.7%) → 8B(61.3%) → 14B(73.3%) → 30B(74.7%) → 32B(80.0%) → VL-32B(82.7%) → Next-80B-A3B(85.3%) → 235B(88.0%) で明確なスケーリング
- **gpt-ossスケーリング(mt=1024)**: 20B(77.3%) → 120B GGUF(90.7%) → 120B MLX(92.0%)
- **gemmaスケーリング**: 12B(54.7%) → 27B(74.7%) → medgemma-27b(76.0%)。医療FTの効果はわずか+1.3%
- **Mistralスケーリング**: small(76.0%) → Large 123B(77.3%) で+1.3%のみ。コスパ悪い
- **モデルサイズと精度**: 1.2B(28%) → 4B(54.7%) → 8B(61.3%) → 14B(73.3%) でサイズとの相関明確

### 2.4 MoE効率性

- **qwen3 MoE比較**: qwen3-next-80b(Active 3B, 85.3%) vs qwen3-235b-a22b(Active 22B, 88.0%) → Active パラメータ7.3倍でわずか-2.7%
- **qwen3.5-27b@8bit(29.5GB)が397b@4bit(223.9GB)と同スコア(87.3%)** → メモリ効率で圧倒的に優位
- **qwen3-next-80b**: 84.67GBで85.3% vs qwen3-32b(34.83GB)で80.0% → メモリ2.4倍で+5.3%

### 2.5 量子化の影響

- 大規模モデル（32B）: -0.5%、合否変わらず → **4bitで十分実用的**
- 中規模モデル（8B）: -4.5%の低下
- 小規模モデル（4B）: -2.3%、セクション間ばらつき大
- 超大規模MoE（397B）: 8bit(89.5%) vs 4bit(87.3%) = -2.3%、合否変わらず
- メモリ削減: 40-47%で一貫
- **GGUF vs MLX比較(mt=1024)**: MLX 8bit(92.0%) > GGUF MXFP4(90.7%) → MLXが+1.3%高いがメモリ2倍(124GB vs 63GB)

### 2.6 Thinkingモデルの知見

- **qwen3のThinking ONは逆効果**: /no_think(88.0%) → Thinking ON(85.3%) で-2.7%、Baselineでは-44.0%壊滅
- `<think>`タグ内の選択肢議論が回答抽出を汚染
- **thinkingモデルにはCoTプロンプトは逆効果**: qwen3.5-397b案B(56.0%)、二重推論で壊滅
- **max_tokensがthinkingモデルの正答率を支配**: nemotron-3-nano/gpt-oss-120B/minimax-m2.5で共通パターン
- **結論**: qwen3には`/no_think`を必ず使用。thinkingモデルには案A(format_strict)が最適

### 2.7 VLモデル vs Non-VL

- 32B: VL(82.7%) > Non-VL(80.0%) で+2.7% → 大規模ではVL有利
- 4B: VL(52.0%) < Non-VL(54.7%) で-2.7% → 小規模では不利

### 2.8 medgemmaの特殊性

- **全セクション評価**: A(76.0%)/B(82.0%)/C(61.3%)/D(76.0%)/E(76.0%)/F(64.0%) → **71.8%不合格（あと13問）**
- セクションAのみ合格、B-Fでは思考モードに入りmax_tokens=50で回答未到達（空回答121/400=30.3%）
- **Few-shotが決定的に有効**: Baseline 17.3%→76.0%（+58.7%）。思考モード抑制効果
- 汎用qwen3-32b(79.3%)に劣る → 医療特化の優位性は限定的

#### medgemma-27b フォーマット・量子化比較（2026-03-13）

| モデル | 量子化 | フォーマット | Section A | 差 | Avg Time |
|---|---|---|---|---|---|
| medgemma-27b MLX Q4_K_S | ~4.6bit | MLX | **76.0%** (57/75) | 基準 | 3.0s |
| gemma-3-27b (汎用) Q4_K_S | ~4.6bit | GGUF | **74.7%** (56/75) | -1.3% | 1.0s |
| medgemma-27b GGUF Q4_K_M | ~4.8bit | GGUF | **70.7%** (53/75) | -5.3% | 0.8s |
| medgemma-27b GGUF Q8_0 | 8bit | GGUF | **69.3%** (52/75) | -6.7% | 0.9s |

**重要な発見**:
- **GGUF Q8_0(69.3%) < GGUF Q4_K_M(70.7%)** — 8bitが4bitより低い逆転現象。量子化レベルは原因ではない
- **MLX(76.0%) >> GGUF Q8_0(69.3%)** — フォーマット差が6.7%の大差を生む
- **GGUF版では医療FT効果が消失**: 汎用gemma-3-27b GGUF(74.7%) > medgemma-27b GGUF Q8_0(69.3%)
- **結論**: MedGemmaの医療FT重みはGGUF変換で劣化する。MLXフォーマットが必須
- **先行研究との整合**: Zhan et al. (2025) 「一部の生物医学FTモデルは量子化/変換に弱い」を支持

### 2.10 一般問題 vs 臨床問題のスケーリング法則の違い（2026-03-13 分析）

全400問評価済み16モデルで一般問題(125問, Sec A+B)と臨床問題(275問, Sec C-F)を比較分析。

- **一般問題**: 対数線形（log-linear）でスケーリング。R²=0.77。14B→400Bで+12pp程度
- **臨床問題**: シグモイド曲線でスケーリング。R²=0.92。閾値付近で急激に改善
- **臨床推論の「創発」**: シグモイド変曲点は約7Bパラメータ付近。8B未満では一般・臨床とも低く差なし、14-80B帯でGap最大（一般が先行）、100B+でGap縮小
- **全モデル平均Gap**: 臨床が一般より約-4.8%低い（paired comparison）
- **Qwen3ファミリー単独分析**: 14モデルで同傾向確認。VLモデルは小型でもGap小さい傾向
- **選択数の交絡分析**: 2+選べ問題で平均-14.5%のペナルティ。ただし臨床Gapの主因は純粋な臨床難易度差（1つ選べのみで比較しても-2.0%）であり、選択数構成差の影響は限定的
- **考察**: 臨床推論は「創発的能力」— 単純な知識再生(一般問題)はスケーリングに対して線形応答するが、多段階推論・情報統合(臨床問題)は一定のモデル規模を超えて初めて機能する
- **プロット**: `plots/general_vs_clinical*.png`, `plots/qwen3_scaling*.png`, `plots/gap_by_size*.png`, `plots/selection_count*.png`

### 7.2 glm-4.7-flash 詳細診断 (2026-02-06)

**問題**: モデルが常に `1.  **Analyze the User's Request:**` から始まる分析モードで出力し、実際の回答(a/b/c/d/e)に到達しない。

|-----------|------|
| System prompt あり/なし | ❌ 変化なし |
| 英語プロンプト | ❌ 変化なし |
| 中国語プロンプト | ❌ 変化なし |
| max_tokens=5〜200 | ❌ 変化なし |
| temperature=0, 0.1 | ❌ 変化なし |
| stop tokens (`\n`, `。`, `、`) | ❌ 変化なし |
| 思考禁止指示 | ❌ 変化なし |
| アシスタントプレフィル | ❌ 変化なし |
